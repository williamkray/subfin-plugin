# Share page: download bundle (ZIP + m3u8)

**status:** closed — shipped in v0.2.5.0

## Feature

Add a "Download" button to the share page (`/Subsonic/share/{uid}`) that downloads a ZIP archive containing:
- All tracks in the share as their original files (preserving original filenames exactly as stored on disk)
- A `playlist.m3u8` file using relative paths so the bundle plays in correct order in any media player that reads files from disk

## UX

On the share page alongside the existing "Download M3U" button:
```
[Download M3U]  [Download ZIP]
```

## Backend endpoint

```
GET /Subsonic/share/{uid}/download
```

- No auth required (share is public by secret)
- Validates `?secret=...` query param against stored share secret (same as share page auth)
- Streams a `application/zip` response with `Content-Disposition: attachment; filename="share-<uid>.zip"`

## ZIP contents

```
share-<uid>.zip
├── playlist.m3u8
├── 03. She's Got a Ticket To Ride [The Beatles] - V0.mp3   ← original filename, unchanged
├── some other track.flac
└── ...
```

File names come directly from `Path.GetFileName(song.Path)` — no renaming, no reformatting.

## m3u8 format

```
#EXTM3U
#EXTINF:duration,Artist - Title
03. She's Got a Ticket To Ride [The Beatles] - V0.mp3
#EXTINF:duration,Artist - Title
some other track.flac
```

- Paths are relative (filename only, no leading slash or directory)
- Order matches the share's track order
- `#EXTINF` duration in seconds (integer, from `RunTimeTicks / 10_000_000`)
- Encoding: UTF-8

## Implementation

Use `System.IO.Compression.ZipArchive` (in-box, no third-party dep):

```csharp
Response.ContentType = "application/zip";
Response.Headers["Content-Disposition"] = $"attachment; filename=\"share-{uid}.zip\"";

using var zip = new ZipArchive(Response.Body, ZipArchiveMode.Create, leaveOpen: true);
var m3u = new StringBuilder("#EXTM3U\n");

foreach (var song in songs)
{
    if (song.Path == null) continue;
    var fileName = Path.GetFileName(song.Path);

    m3u.Append($"#EXTINF:{(int)(song.RunTimeTicks / 10_000_000)},{song.AlbumArtists.FirstOrDefault() ?? ""} - {song.Name}\n");
    m3u.Append(fileName + "\n");

    var entry = zip.CreateEntry(fileName, CompressionLevel.NoCompression);
    using var entryStream = entry.Open();
    using var fileStream = System.IO.File.OpenRead(song.Path);
    await fileStream.CopyToAsync(entryStream);
}

var m3uEntry = zip.CreateEntry("playlist.m3u8");
using var m3uStream = new StreamWriter(m3uEntry.Open(), Encoding.UTF8);
await m3uStream.WriteAsync(m3u.ToString());
```

## Notes

- Use `CompressionLevel.NoCompression` for audio files (already compressed; no size benefit from recompressing)
- The endpoint lives in `WebController` (not SubsonicController) since it's under `/Subsonic/`
- Share secret validation: reuse the same lookup pattern as the share page (`SubsonicStore.GetShareByUid`)
- If two tracks share the same filename (different directories), they'll collide in the ZIP — acceptable edge case for now; can suffix with index if needed later
- Streams directly to response body — no temp file needed

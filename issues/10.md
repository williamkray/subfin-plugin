# Plugin artifact distribution

**status:** implemented (quick-and-dirty via static hosting)

## Current solution

`./scripts/publish.sh` — builds, zips, generates manifest, and uploads to `box`.

**Plugin repo URL** (add to Jellyfin → Dashboard → Plugins → Repositories):
```
https://home.kray.pw/jellyfin-plugin-subsonic-manifest.json
```

Artifacts:
- `https://home.kray.pw/jellyfin-plugin-subsonic-manifest.json` — Jellyfin repo manifest
- `https://home.kray.pw/jellyfin-plugin-subsonic_<version>.zip` — plugin ZIP

## What the script does

1. `dotnet publish -c Release`
2. Writes a proper `meta.json` (with `assemblies`, `status: Active`) into a temp dir
3. ZIPs `Jellyfin.Plugin.Subsonic.dll` + `meta.json`
4. Computes MD5 checksum
5. Generates `manifest.json` (Jellyfin repo format) with checksum + source URL
6. SCPs both files to `box:/opt/services/home-site/site/`

## Future

Replace with GitHub Actions + GitHub Pages + Releases when a proper CI/CD process is set up. The manifest format and artifact names will stay the same.

## Notes

- Only keeps the latest version in the manifest (no version history). Jellyfin will offer the latest to all users.
- `version` in manifest is `meta.json version` + `.0` suffix (Jellyfin expects 4-part version string, e.g. `0.1.0.0`)
- Requires `jq`, `zip`, `md5sum` on the local machine
- SSH to `box` uses preconfigured SSH config — no credentials needed

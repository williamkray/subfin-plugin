# Bookmarks: getBookmarks / createBookmark / deleteBookmark

**status:** won't fix — out of scope

## Problem

Bookmark endpoints are missing from the dispatch table. Clients that use bookmarks (e.g. for audiobook position tracking) will get "Unknown method" errors.

## OpenSubsonic spec

**getBookmarks** — returns all bookmarks for the user
```xml
<bookmarks>
  <bookmark position="12345" username="user1" comment="..." created="..." changed="...">
    <entry id="..." title="..." .../>  <!-- song entry -->
  </bookmark>
</bookmarks>
```

**createBookmark** — params: `id` (song), `position` (ms), `comment` (optional)

**deleteBookmark** — params: `id` (song)

## Jellyfin mapping

Jellyfin stores playback position in `UserData.PlaybackPositionTicks` (100-nanosecond ticks).

```csharp
// Read
var data = _userData.GetUserData(user, item);
long positionMs = data.PlaybackPositionTicks / 10000;

// Write (createBookmark)
data.PlaybackPositionTicks = positionMs * 10000;
_userData.SaveUserData(user, item, data, UserDataSaveReason.PlaybackProgress, CancellationToken.None);

// Delete (deleteBookmark)
data.PlaybackPositionTicks = 0;
_userData.SaveUserData(user, item, data, UserDataSaveReason.PlaybackProgress, CancellationToken.None);
```

## getBookmarks implementation sketch

Query all items with a non-zero PlaybackPositionTicks for the user:
```csharp
var query = new InternalItemsQuery(user)
{
    IncludeItemTypes = [BaseItemKind.Audio],
    Recursive = true,
};
var items = _library.GetItemList(query)
    .Where(i => _userData.GetUserData(user, i).PlaybackPositionTicks > 0)
    .ToList();
```

Note: this is potentially slow on large libraries. Consider returning only items the client has explicitly marked (no Jellyfin native bookmark concept beyond position — every played item may have a non-zero position).

## Response fields

- `position` — ms (from PlaybackPositionTicks / 10000)
- `username` — Subsonic username
- `comment` — no native Jellyfin equivalent; can store in DB or omit
- `created` / `changed` — can use `UserData.LastPlayedDate` or current time
- `entry` — full song child (same shape as `<song>` in other responses)

## Notes

- Jellyfin doesn't have a separate "bookmark" concept distinct from playback position, so createBookmark / deleteBookmark simply set/clear the position.
- Comment field is unsupported natively; either ignore it or add a `bookmarks` table to the SQLite store.
- Validate against navic-sme and dsub2000-sme before implementing response shape.

# Transcoding: stream endpoint ignores bitRate/format params

**status:** resolved

## Problem

The `stream` endpoint serves files directly at full quality. Clients that request a lower bitrate (mobile data saving, bandwidth-limited setups) or a specific format receive the original file regardless.

Current implementation:
```csharp
return PhysicalFile(item.Path, ItemMapper.AudioMimeType(item.Container), null, true);
```

Params `bitRate`, `maxBitRate`, `format`, `estimateContentLength`, `timeOffset` are not read.

## Also: download content-type bug

`download` endpoint hardcodes `"audio/mpeg"` regardless of actual container:
```csharp
return PhysicalFile(item.Path, "audio/mpeg", Path.GetFileName(item.Path), true);
```
Fix: use `ItemMapper.AudioMimeType(item.Container)`.

## OpenSubsonic spec

- `bitRate` — kbps cap (e.g. 128, 192, 320)
- `maxBitRate` — alternate cap param (same semantics)
- `format` — target format: `mp3`, `flac`, `ogg`, `opus`, `aac`, `raw` (raw = no transcode)
- `timeOffset` — seconds to seek into the file before streaming
- `estimateContentLength` — hint to set Content-Length header

## Approach options

### Option A: Redirect to Jellyfin's own transcoding (preferred)
Jellyfin already has an audio transcoding pipeline. Redirect to:
```
/Audio/{itemId}/universal?...transcode params...
```
This avoids reimplementing ffmpeg in the plugin and reuses Jellyfin's existing transcode logic.

Pros: Works immediately, no ffmpeg dependency in plugin.
Cons: Requires mapping Subsonic params → Jellyfin audio stream params; may need auth token in redirect.

### Option B: In-process ffmpeg via process spawn
Spawn `ffmpeg` inside the plugin, pipe output to response stream. Matches how subfin likely does it.
Pros: Full control.
Cons: Complex, long-running request handling, error-prone.

## Recommended first step

1. Fix the `download` content-type bug (trivial, one line)
2. Parse `bitRate`/`format` params and check if transcoding is requested
3. If `format == "raw"` or no bitrate constraint → serve direct (current behavior)
4. Otherwise → investigate Jellyfin's `/Audio/{id}/universal` endpoint signature and redirect

## QueryParams to add
```csharp
public int? BitRate => TryGetInt("bitRate") ?? TryGetInt("maxBitRate");
public string? TranscodeFormat => Get("format");
public int? TimeOffset => TryGetInt("timeOffset");
```

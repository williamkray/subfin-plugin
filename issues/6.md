# Stubbed / missing endpoint audit

**status:** audit complete — see linked issues for individual work items.

## Audit findings (2026-03-27)

The controller has 44 dispatched endpoints. The following gaps were identified:

### Stubs (return empty OK)
- `getLyrics` → empty `<lyrics/>` — see issue #2
- `getLyricsBySongId` → empty `structuredLyrics[]` — see issue #2

### Missing from dispatch (return "Unknown method")
Grouped by realistic priority for this audio-focused plugin:

**Implement:**
- `getBookmarks` / `createBookmark` / `deleteBookmark` — see issue #22
- `startScan` — see issue #23
- Transcoding params on `stream` (`bitRate`, `maxBitRate`, `format`) — see issue #21

**Intentionally out of scope (audio-only plugin):**
- `hls` — HTTP Live Streaming
- `jukeboxControl` — remote jukebox mode
- `getPodcasts` / `getNewestPodcasts` / `refreshPodcasts` / `createPodcastChannel` / `deletePodcastChannel` / `downloadPodcastEpisode`
- `getInternetRadioStations` / `createInternetRadioStation` / `updateInternetRadioStation` / `deleteInternetRadioStation`
- `getVideos` / `getVideoInfo` / `getCaptions`
- `getChatMessages` / `addChatMessage`

**Deferred (user management — needs policy decision):**
- `createUser` / `updateUser` / `deleteUser` / `changePassword`
- `getUsers` is currently aliased to `getUser` (single user); returning multi-user list requires admin check

### `getScanStatus` partial stub
Returns `scanning=false, count=0` hardcoded. Will be upgraded when `startScan` is implemented (issue #23).

### Stream — no transcoding
`stream` endpoint serves the file at full bitrate via `PhysicalFile`. `bitRate`, `maxBitRate`, `format`, `estimateContentLength` params are silently ignored. See issue #21.

### Download — minor bug
`download` hardcodes `ContentType = "audio/mpeg"` regardless of actual file format. Should use `ItemMapper.AudioMimeType(item.Container)`.

# Lyrics not returned (getLyrics / getLyricsBySongId)

**clients:** Tempus, DSub2000, and others.

**status:** open — actively investigating with localenv lyrics plugin.

## Problem

Both `getLyrics` and `getLyricsBySongId` return empty stubs. No lyrics are surfaced to clients even when lyrics exist on the server.

## Current behavior

```csharp
// getLyrics
w.WriteStartElement("lyrics", Ns); w.WriteEndElement();

// getLyricsBySongId
{"lyricsList": {"structuredLyrics": []}}
```

## Setup note

The Jellyfin lyrics plugin is now installed in localenv for accurate testing. Previously, lyrics were unavailable in Jellyfin itself, so the stub behavior was undetectable.

## What needs implementing

**getLyrics** (legacy, line-only):
- Param: `artist`, `title` (used for matching — not reliable; prefer looking up by song ID via `id` param if clients send it)
- Response: `<lyrics artist="..." title="...">line 1\nline 2\n...</lyrics>`
- Jellyfin API: look up `Audio` item, then check `ILyricManager.GetLyrics(item)` or similar

**getLyricsBySongId** (OpenSubsonic structured):
- Param: `id` (song ID)
- Response: `structuredLyrics[]` with `lang`, `synced` (bool), `line[]` (`{ start?: ms, value: string }`)
- Jellyfin API: same — `ILyricManager` returns `LyricDto` with `Lyrics[]` containing `Start` (ticks) and `Text`
- Convert `Start` ticks → milliseconds (`ticks / 10000`)
- If `Start` is set → `synced: true`; omit `start` field if not synced

- Check namespace: likely `MediaBrowser.Controller.Lyrics` or similar
- Probe API: `./scripts/probe-api.sh` to verify signature

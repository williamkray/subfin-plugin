# Lyrics not returned (getLyrics / getLyricsBySongId)

**clients:** Tempus, DSub2000, and others.

**status:** closed — shipped in v10.11.5.0.

## Resolution

Both endpoints now call `ILyricManager.GetLyricsAsync(audio, ct)` from `MediaBrowser.Controller.Lyrics`.

**getLyrics:**
- Looks up by `id` param (preferred); falls back to title/artist search
- Returns `<lyrics artist="..." title="...">line1\nline2\n...</lyrics>`

**getLyricsBySongId:**
- Looks up by `id` (required)
- Returns `structuredLyrics[]` with `lang="und"`, `synced` (from `LyricMetadata.IsSynced`), and `line[]`
- `LyricLine.Start` ticks → milliseconds via `/ 10000`

**API facts (Jellyfin 10.11.6):**
- `ILyricManager` namespace: `MediaBrowser.Controller.Lyrics`
- Method: `GetLyricsAsync(Audio item, CancellationToken ct)` → `Task<LyricDto?>`
- `LyricDto.Lyrics`: `IReadOnlyList<LyricLine>` (`Text: string`, `Start: long?` in ticks)
- `LyricDto.Metadata`: `LyricMetadata` with `IsSynced: bool?`, `Artist`, `Title`, etc.
- `IsSynced` is null even for `.lrc` files — detect sync by checking all lines have `Start > 0`

**Client compat notes:**
- Navic: `displayArtist`, `displayTitle` non-nullable — always include; `line.start` non-nullable `Int` — always include (0 for unsynced)
- Tempus: uses `getLyricsBySongId` only when `songLyrics` is in `getOpenSubsonicExtensions`; auto-unboxes `line.start` in seek handler — crash if synced line missing start
- `songLyrics` added to extensions list so Tempus uses the scrolling UI path
